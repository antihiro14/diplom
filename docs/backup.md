# Резервне копіювання проєкту

## 1. Мета

Цей документ описує рекомендації щодо резервного копіювання проєкту аналізу їжі за фотографією.

## 2. Стратегія резервного копіювання

Для проєкту рекомендується комбінована стратегія резервного копіювання:

- повна резервна копія — повне копіювання всього проєкту;
- інкрементальна копія — збереження лише змінених файлів після останньої копії;
- диференціальна копія — збереження змін відносно останньої повної копії.

На поточному етапі основним способом резервного копіювання є:
- Git-репозиторій;
- локальні архівні копії;
- резервне збереження моделей та результатів навчання.

## 3. Частота створення резервних копій

Рекомендується така частота:

- після кожної важливої зміни коду — Git commit + Git push;
- після завершення етапу навчання моделі — окрема локальна резервна копія;
- раз на тиждень — повний архів проєкту;
- перед оновленням або великими змінами — додаткова резервна копія.

## 4. Зберігання та ротація копій

Резервні копії слід зберігати:

- у GitHub-репозиторії;
- локально на комп’ютері;
- за можливості — у хмарному сховищі.

Ротація копій:
- зберігати щонайменше 3 останні повні резервні копії;
- старі проміжні копії можна видаляти після перевірки нових;
- окремо зберігати важливі версії моделей.

## 5. Що потрібно резервно копіювати

### 5.1 Вихідний код
Увесь код проєкту:
- `main.py`
- `src/`
- `models/`
- `docs/`
- `README.md`
- `requirements.txt`

### 5.2 Бази даних
На поточному етапі база даних у проєкті відсутня.  
У майбутньому, якщо буде додано базу даних, до резервної копії мають входити:
- структура бази;
- дані користувачів;
- журнали змін.

### 5.3 Конфігураційні файли
Необхідно зберігати:
- `requirements.txt`
- файли конфігурації середовища
- налаштування запуску
- конфігурації Docker / CI/CD

### 5.4 Користувацькі дані
На поточному етапі користувацькі дані не накопичуються.  
У майбутньому слід резервно копіювати:
- завантажені користувачами зображення;
- результати аналізу;
- історію прогнозів.

### 5.5 Логи системи
Якщо у проєкті ведуться логи, необхідно зберігати:
- журнали запуску;
- журнали помилок;
- результати навчання моделі.
