# Тести як жива документація

У проєкті тести розглядаються не лише як інструмент перевірки правильності роботи коду, але і як приклади використання окремих компонентів системи.

Такий підхід дозволяє:
- показати очікувану поведінку функцій;
- продемонструвати формат вхідних і вихідних даних;
- спростити розуміння коду при подальшій розробці.

## План подальшого використання тестів

У наступних етапах розробки планується додати тести для:
- перевірки завантаження датасету;
- перевірки попередньої обробки зображень;
- перевірки коректності навчального циклу;
- перевірки форми вихідних даних моделі;
- перевірки допоміжних функцій.

## Значення для документації

Наявність тестів дає можливість використовувати їх як приклади практичного використання модулів системи, тобто як елемент "живої" документації.
