# Архітектура проєкту

## Загальна структура

Проєкт призначений для аналізу їжі за фотографією з використанням методів комп’ютерного зору та машинного навчання. На поточному етапі система має модульну структуру і складається з кількох основних частин.

### Основні модулі

- `main.py` — головний файл запуску проєкту;
- `src/dataset.py` — модуль для роботи з датасетом зображень;
- `src/train.py` — модуль навчання моделі;
- `src/utils.py` — допоміжні функції для обробки даних та сервісних операцій;
- `models/classifier.py` — модуль з описом архітектури класифікатора.

## Призначення компонентів

### main.py
Використовується як точка входу в програму. Координує запуск основних сценаріїв роботи проєкту.

### src/dataset.py
Містить логіку завантаження, підготовки та доступу до зображень і міток класів. Відповідає за роботу з навчальними даними.

### src/train.py
Містить реалізацію процесу навчання моделі: проходження епох, обчислення функції втрат, оновлення параметрів та оцінювання якості.

### src/utils.py
Містить допоміжні функції, які використовуються в різних частинах проєкту, наприклад для роботи з файлами, зображеннями чи збереженням результатів.

### models/classifier.py
Містить опис моделі класифікації, яка використовується для розпізнавання класу страви за зображенням.

## Взаємодія компонентів

Робота системи виконується послідовно:

1. Дані завантажуються з датасету.
2. Зображення проходять попередню підготовку.
3. Підготовлені дані передаються у модель класифікації.
4. У процесі навчання модель оновлює параметри.
5. Результати можуть бути використані для подальшого тестування та інтеграції в застосунок.

## Причини вибору архітектури

Обрана модульна архітектура дозволяє:
- розділити відповідальність між частинами системи;
- полегшити супровід і розвиток проєкту;
- спростити документування коду;
- підготувати основу для подальшого розширення функціональності.

## Подальший розвиток

У майбутньому структура проєкту може бути розширена за рахунок:
- модуля оцінювання моделі;
- модуля прогнозування для нових зображень;
- модуля визначення інгредієнтів;
- модуля оцінки харчової цінності страви;
- інтеграції з мобільним або веб-застосунком.
