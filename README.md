# Food Recognition System

## Опис
Цей проєкт є частиною бакалаврської роботи та спрямований на розпізнавання страв за зображеннями з використанням методів комп'ютерного зору.

## Функціональність
- Обробка датасету зображень страв
- Навчання моделі класифікації
- Визначення типу страви за фото

## Структура
- `src/` — логіка обробки даних та навчання
- `models/` — архітектура моделі
- `data/` — приклади даних (без повного датасету)
- `main.py` — точка входу

## Стан проєкту
Проєкт знаходиться на початковому етапі розробки.

## Документування коду

У проєкті використовується документування коду на основі `docstring` для Python-модулів, функцій, класів і методів.

### Що потрібно документувати

Обов’язковому документуванню підлягають:

- усі публічні функції;
- класи та їх методи;
- модулі з ключовою логікою;
- частини коду, що реалізують важливі етапи обробки даних;
- алгоритми, які можуть бути складними для розуміння.

### Що має містити документація

Кожен docstring повинен містити:

- короткий опис призначення;
- опис параметрів;
- опис результату роботи;
- можливі винятки або обмеження;
- за потреби — примітки щодо використання.

### Додаткова документація

У папці `docs/` розміщуються:

- `architecture.md` — опис архітектури проєкту;
- `business_logic.md` — опис бізнес-логіки;
- `generate_docs.md` — інструкція з генерації документації;
- `generated/` — згенерована HTML-документація.

## DevOps документація

У папці docs знаходяться інструкції:

- deployment.md — запуск
- update.md — оновлення
- backup.md — резервне копіювання
### Генерація документації
